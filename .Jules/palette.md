## 2024-05-18 - Hover-Revealed UI Actions Missing Keyboard Accessibility & ARIA labels
**Learning:** Found that UI actions designed to be revealed only on hover (using classes like `group-hover:opacity-100`) are completely inaccessible to keyboard users because they never receive focus or become visible during tab navigation. Additionally, multiple icon-only buttons across chat components were missing `aria-label`s, rendering them ambiguous to screen readers.
**Action:** When implementing hover-revealed UI patterns (e.g., "Edit", "Delete" buttons), always pair the hover classes with `focus-visible:opacity-100 focus-visible:ring-2 focus-visible:outline-none` to ensure they can be discovered and activated via keyboard. Always provide descriptive `aria-label`s for all icon-only buttons.
